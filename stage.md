# Реестр изменений (Stage) - 14.04.2026

В ходе текущего этапа работ были произведены исправления в интерфейсе, оптимизация производительности и настройка системы автоматического обновления данных филиалов и работы камер.

## 1. Интерфейс управления филиалами
- **Исправление модального окна**: Для карточки редактирования/создания филиала добавлены свойства `max-height` и `overflow-y-auto`. Теперь на экранах с низким разрешением все поля (включая IP и RTSP) доступны для прокрутки.
- **Кнопка синхронизации**: В заголовок страницы управления филиалами добавлена кнопка «Синхронизировать».

## 2. Синхронизация данных (IP и RTSP)
- **Переход на Server Actions**: Логика обновления данных из Excel-отчета перенесена из внешнего скрипта в основное приложение. Это решило проблему отсутствия модулей Prisma в Docker-контейнере.
- **Оптимизация деплоя**: Удалена проблемная команда `docker exec` из файла `deploy.bat`. Процесс деплоя стал стабильнее.
- **Файл отчета**: Файл `camera_report.xlsx` перемещен в папку `scripts/` и включен в сборку проекта.

## 3. Работа с камерами (FFmpeg)
- **Системная установка**: В `Dockerfile` добавлена команда установки `ffmpeg` прямо в операционную систему контейнера (Debian). Это гарантирует стабильный доступ к видео-движку.
- **Динамические пути**: Код в `src/lib/audit.ts` теперь автоматически определяет ОС (Windows или Linux) и находит правильный исполняемый файл.
- **Фоновый захват**: Процесс создания скриншотов теперь полностью асинхронный. Открытие или закрытие смены происходит мгновенно, не дожидаясь ответа от камеры.
- **Исправление ошибок**: Устранен конфликт параметров (ошибка `Unrecognized option 'stimeout'`), возникавший на серверной версии FFmpeg.

## 4. Система логирования и отладки
- **Раздел «Логи камер»**: Создана новая административная страница `/admin/logs`, доступная из раздела «Аналитика и Отчёты».
- **Функционал логов**: Реализована возможность просмотра последних 1000 строк вывода FFmpeg и кнопка полной очистки логов для удобства отладки новых камер.
- **Action-логирование**: Добавлены подробные статусы: `[CAPTURE]`, `[SUCCESS]`, `[FAILURE]`, `[TIMEOUT]`, что позволяет быстро выявить проблемы с сетью или авторизацией на камерах.

---
**Статус**: Изменения применены и готовы к финальной проверке после запуска `deploy.bat`.
